# Cantina findings — test-coverage audit

Audited on branch `debug/pr94-db-lock` (2026-07-04). Method: inventory of every
finding ID referenced in `src/`, `scripts/`, `docs/` (`Cantina #N`, `MA#N`,
`Finding #N`, and the self-review aliases `X*`, `S*`, `R*`, `B*`, `C*`), then
for each finding: locate the fix, enumerate the unit tests that pin it
(`cargo test --lib -- --list`, 275 tests; ~100 reference a finding ID by name),
and map e2e coverage in `scripts/e2e-*.sh` / `scripts/verify-event-completeness.sh`.

Coverage classes:

- **UNIT+E2E** — dedicated unit tests *and* an e2e script that exercises the fixed path.
- **UNIT** — dedicated unit tests only.
- **PARTIAL** — tests exist but miss a material branch of the fix (detailed per row).
- **NONE** — no test names or assertions target the fix.
- **FIX-ON-BRANCH** — fix + tests live on an unmerged branch.

Honesty rule applied throughout: an assertion that merely *passes through* the
fixed code (e.g. `test_pgstore_logs` executing the S3 atomic counter on the
happy path) is **not** counted as coverage of the fix.

Note on ID collisions: the numbering was reused across review rounds, so
"Cantina #5" (BURN serial collision) is distinct from "Finding #5" (synthetic
projector redesign), and "Cantina #10" (BridgeEvent metadata ABI encoding) is
distinct from "Finding #10" (atomic faucet registration). Both members of each
pair are listed.
