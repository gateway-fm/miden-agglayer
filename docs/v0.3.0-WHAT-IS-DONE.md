## Bali-specific caveat (read before deploying)

Bali's CURRENT pod has accounts that were created Private (pre-`dbe5c2d`-fix). The v0.3.0 runtime self-heal CANNOT recover them — `import_account_by_id` returns `AccountIsPrivate` on Private accounts. So **deploying v0.3.0 to bali in place will NOT cure marti's stuck deposit**.

Bali needs a **full redeploy** with fresh Public accounts + aggkit reconfig pointing at the new account IDs. Procedure in `docs/REDEPLOY_RUNBOOK_BALI.md`. Three-step summary:

1. Roll the proxy with `--init --reset-miden-store` (gets a clean Public-mode deployment).
2. Update aggkit's config to point at the new `bridge_accounts.toml` IDs.
3. Restart aggkit. Next aggoracle push hits the new proxy, GER injects cleanly, marti's deposit flips ready.

All future deployments (and any other proxy redeployed off v0.3.0) self-heal on this bug class without operator intervention.
